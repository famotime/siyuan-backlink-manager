# 关联的定义块说明

## 含义

在本插件里：

- 定义块：当前文档中被引用的内容块，例如块 B
- 反链块：引用了定义块的内容块，例如块 A
- 关联的定义块：反链块所在路径、父级路径或子级路径中，引用到的其他文档的内容块，例如块 C

可以直接理解为：

当你正在查看当前文档里的定义块 B 的反链时，如果某条反链 A 除了引用 B 之外，还引用了其他文档里的某个块 C，那么这个块 C 就是“关联的定义块”。

补充说明：

- “关联的定义块”显示的是定义块本身的文本内容，不是双链锚文本
- 从实现上看，“关联的定义块”指的是非当前文档的定义块

## 例子 1：反链块自己引用了其他文档

假设：

- 文档 1 中有块 B：`反链面板设计`
- 文档 2 中有块 C：`筛选逻辑说明`
- 文档 3 中有块 A：`这里要同时参考 ((块B)) 和 ((块C))`

现在进入文档 1 查看块 B 的反链：

- 块 B 是“当前文档定义块”
- 块 A 是“反链块”
- 块 C 是“关联的定义块”

原因是：块 A 在引用块 B 的同时，也引用了另一个文档中的块 C。

## 例子 2：父级路径里存在其他引用

假设反链内容处在一个父级结构下：

```md
项目说明
- 参考：((块B))
- 补充：((块C))
```

如果插件设置中启用了“查询父级关联定义块”，那么你在查看块 B 的反链时，父级路径中的块 C 也会被识别为“关联的定义块”。

## 例子 3：子级路径里存在其他引用

假设反链块是标题或列表项，并且它的子级中存在其他引用：

```md
## 实现方案 ((块B))
这里的补充材料见：((块C))
```

或者：

```md
- 方案说明：((块B))
  - 补充资料：((块C))
```

如果插件设置中启用了“查询标题下关联定义块”或“查询列表项下关联定义块”，那么这些子级中的块 C 也会被识别为“关联的定义块”。

## 一句话总结

“关联的定义块”就是：某条反链在指向当前文档定义块的同时，还关联到的其他文档中的定义块。它主要用于在筛选面板中进一步过滤反链内容。
