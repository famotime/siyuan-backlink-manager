# 反链上下文基线与术语表

## 文档目的

本文用于冻结当前迭代的术语和代码基线。

正式规则定义见：

- `docs/上下文状态渐进显示规则.md`

本文回答两个问题：

- 当前代码实际上怎么做
- 当前代码与正式规则还差在哪里

## 术语表

| 术语 | 定义 |
| --- | --- |
| 定义块 | 当前文档中被引用的内容块 |
| 反链块 | 引用了定义块的内容块 |
| 焦点块 | 当前这条反链真正落在的块 |
| 锚点单元 | 当前档位中必须展示的最小原文单元 |
| 结构壳 | 为了定位结构而保留的最小壳层，例如列表项标题壳 |
| 相邻单元 | 与锚点同一节内、同一层级优先、文档顺序最近的前后语义单元 |
| 所属标题节 | 由当前归属标题开始、到下一个同级或更高级标题前结束的范围 |
| 上下文层级 | `核心 / 近邻 / 扩展 / 全文` 四层正文范围 |
| 命中来源 | 当前结果为何命中的上下文来源，例如 `self`、`parent`、`sibling_prev` |
| 命中摘要 | 对命中原因的短说明，不替代原文正文 |
| 预算 | 对可见上下文规模的限制，包括片段数、字符数、深度、展开规模 |

## 当前实现基线

## 一、范围判定

### 1. 核心

当前实现：

- 普通块：显示当前块
- 列表场景：若焦点块位于列表项内，则把整个列表项子树作为 `core`

与正式规则的差异：

- 正式规则要求列表 `core` 以“结构壳 + 焦点子块”为主
- 当前实现的列表 `core` 偏大，容易把无关子项一起带入

涉及模块：

- `src/service/backlink/backlink-source-window.js`
- `src/components/panel/backlink-document-interaction.js`

### 2. 近邻

当前实现：

- 普通块：优先按文档顺序取前后相邻块
- 列表场景：优先取前后同级列表项，并把相邻列表项的整棵子树带入窗口

与正式规则的差异：

- 正式规则要求近邻优先保持在同一节内
- 正式规则要求邻居父项默认仅展示结构壳与首层直接子项，不应直接带入整棵深层子树

涉及模块：

- `src/service/backlink/backlink-source-window.js`
- `src/service/backlink/backlink-preview-assembly.js`
- `src/components/panel/backlink-protyle-rendering.js`

### 3. 扩展

当前实现：

- 向上找到最近一个标题
- 向下找到下一个任意标题前
- 如果文档无标题，则直接退化为全文

与正式规则的差异：

- 正式规则要求按“所属标题节”判定下边界，即下一个同级或更高级标题前
- 正式规则要求无标题文档先尝试结构容器兜底，而不是直接全文

涉及模块：

- `src/service/backlink/backlink-source-window.js`

## 二、上下文来源

当前显式来源：

- `self`
- `document`
- `parent`
- `child_headline`
- `child_list`
- `sibling_prev`
- `sibling_next`
- `expanded`

当前问题：

- 来源模型已经统一，但“来源片段”和“正文范围”还不是一套完全一致的规则
- `expanded` 更像宽泛扩展兜底，还没有直接表达“所属标题节”与“结构壳折叠”这两个正式概念

涉及模块：

- `src/service/backlink/backlink-context-rules.js`
- `src/service/backlink/backlink-context.js`
- `src/service/backlink/backlink-preview-assembly.js`

## 三、渲染方式

当前实现同时存在两条局部渲染路径：

- source window 裁剪渲染
- fragment/preview 组装渲染

当前问题：

- 列表与普通块在不同档位走不同渲染分支，规则理解成本较高
- “结构壳可见但深层默认折叠”的能力还不够明确

涉及模块：

- `src/components/panel/backlink-document-interaction.js`
- `src/components/panel/backlink-protyle-rendering.js`

## 四、交互状态

当前实现已具备：

- 文档级 `core / nearby / extended / full` 状态切换
- 文档折叠状态保存
- 列表折叠和标题 more 展开状态同步

当前缺口：

- 当前层级与新暴露内容的反馈仍偏弱
- 编辑或拖拽后“按当前档位重新计算范围”的契约未正式写清

涉及模块：

- `src/components/panel/backlink-document-view-state.js`
- `src/components/panel/backlink-panel-header.js`

## 当前结论

当前实现已经具备四层状态、来源片段、预算控制和局部渲染基础。

下一轮需要收敛的，不是“再加更多来源”，而是把以下三件事对齐：

1. 正文范围判定与正式规则对齐
2. 列表场景的结构壳渲染与折叠能力补齐
3. 渲染路径和状态反馈对用户保持一致
