# 反链关键字筛选匹配范围说明

## 含义

反链面板顶部的关键字输入框，用来筛选当前展示的反链块。

它默认匹配的不是“仅反链块正文”，而是整条反链可查询的上下文内容。

## 默认匹配范围

输入普通关键字时，匹配范围包括：

- 反链块自身的文本内容
- 反链所在文档的信息
  - 包括文档名
  - 也包括文档的 `markdown`、`content`、`name`、`alias`、`memo`、`tag`
- 反链块的父级路径文本
- 如果反链块是标题块，则包括标题下的子级文本
- 如果反链块位于列表项中，则包括该列表项下的子级文本

可以直接理解为：

关键字会在“这一条反链相关的整段上下文”中查找，而不是只查找反链块当前这一行。

## 锚文本匹配语法

搜索框支持以下语法：

- 普通关键字：匹配内容
- `%关键词`：只匹配锚文本内容
- `-关键词`：排除内容
- `-%关键词` 或 `%-关键词`：排除锚文本内容

补充说明：

- 这里的“内容”是指反链块所有可查询的文本内容，包含文档名及上下文
- 这里的“锚文本内容”仅指反链块中用作块引用锚文本的文本内容
- 因此“内容”的范围大于“锚文本内容”

## 例子

假设某条反链在面板中的上下文如下：

```md
文档名：项目设计说明

## 反链背景
这里需要参考 ((块B))，详细规则见 ((块C))
```

那么：

- 搜索 `项目设计`，可以匹配到这条反链，因为文档名会参与匹配
- 搜索 `详细规则`，可以匹配到这条反链，因为反链块正文会参与匹配
- 搜索 `%块B`，只会检查锚文本里是否包含 `块B`
- 搜索 `-块C`，会排除内容中包含 `块C` 的反链

## 一句话总结

反链面板中的关键字筛选，默认匹配的是整条反链的上下文内容；只有使用 `%` 前缀时，才会收窄为仅匹配锚文本。
