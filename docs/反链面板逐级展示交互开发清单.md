# 反链面板逐级展示交互开发清单

## 文档目的

本文将《反链面板逐级展示检查与交互优化建议》转为可执行的开发工作清单，用于后续按步骤实现逐级展示交互优化。

本文重点约束一项界面原则：

- 当前标题区信息和按钮已经较密集
- 新的逐级展示交互，不继续堆叠到标题行
- 新交互统一在标题下方新增一个独立组件区域承载

## 设计约束

### 1. 标题行保持轻量

当前标题行已经承担以下职责：

- 文档折叠/展开
- 文档标题展示
- 命中来源标签
- 命中摘要
- 同文档反链切换

因此后续不建议继续在标题行追加：

- 新按钮组
- 多段状态文本
- 多枚 badge
- 引导型提示文案

### 2. 新交互放到标题下方组件区

建议在每个文档标题行下方增加一个独立的“上下文控制条”组件区域，承担以下职责：

- 显示当前上下文层级
- 提供上下文层级切换入口
- 显示本次层级下新增的上下文类型
- 显示预算裁剪提示
- 提供轻量提示信息

该区域应与标题行视觉分层，避免用户把“折叠文档”和“展开上下文”混为一谈。

### 3. 优先提升可发现性

本轮交互优化的首要目标不是增加更多能力，而是让已有能力更容易被发现、理解和验证。

## 目标产出

完成后，反链面板应具备以下体验：

- 用户无需记忆隐藏手势，也能看到“上下文可以逐级切换”
- 用户能明确知道当前处于 `核心 / 近邻 / 扩展 / 全文` 的哪一层
- 用户能理解本次展开新增了什么上下文
- 用户能分辨“没有更多上下文”和“有上下文但被裁剪”两种情况
- 标题行信息密度不再继续上升

## 建议组件形态

新增组件建议命名为：

- `backlink-context-control-row`

建议位于：

- 每个文档标题行下方
- 具体内容渲染区上方

建议包含的子区域：

1. 层级状态区

- 显示当前层级：`核心` / `近邻` / `扩展` / `全文`
- 可以用 pill、segmented tag 或轻量按钮呈现

2. 层级切换区

- 至少提供一个显式入口，例如：
  - `展开上下文`
  - `下一层`
  - 当前层级按钮本身即可点击推进

3. 结果说明区

- 展示当前新增内容，例如：
  - `已显示：前后相邻块、父级`
  - `已显示：标题子级`
  - `已进入全文`

4. 裁剪提示区

- 当预算生效时显示：
  - `部分上下文已裁剪`
  - `继续展开查看更多`

## 开发工作包

以下工作包按优先级排序，建议逐项开发，每项都遵守现有工作流：

- 先补测试
- 再实现
- 完成后验证
- 单独提交

## 实施原则

后续开发时建议始终坚持以下原则：

- 不把新能力继续塞进标题行
- 先解决可发现性，再解决精细提示
- 控制条文案必须短，不做长说明
- 状态信息优先，其次才是引导信息
- 同一文档内反链切换时，控制条状态必须同步更新
- 所有新增交互都应保留原有快捷行为兼容

## 完成定义

本清单完成后，应达到以下状态：

- 用户在不阅读文档的情况下，也能发现逐级展示入口
- 用户能明确知道当前层级
- 用户能理解本次增加了哪些上下文
- 用户能区分“没有更多内容”和“内容被裁剪”
- 标题行不再继续增压，新增交互统一下沉到标题下组件区
