# 反链面板逐级展示检查与交互优化建议

## 文档目的

本文用于回答两个实际问题：

1. 本轮优化后，为什么反链面板首屏看起来变化不大
2. 逐级展示虽然已实现，但在交互上是否还需要更明显的操作方式

本文不针对具体代码改动，只从当前实际行为和用户体验出发，给出检查方法与优化建议。

## 一、为什么看起来“变化不大”

当前版本的优化重点，主要是把上下文能力统一为一套规则，而不是把首屏默认内容直接扩充很多。

本轮已经落地的核心变化主要包括：

- 统一上下文片段模型
- 统一搜索、筛选、展示的上下文来源
- 增加命中来源标签与命中摘要
- 建立 `核心 / 近邻 / 扩展 / 全文` 四层展示策略
- 支持文档级渐进展开
- 增加上下文预算控制
- 增加 `紧凑 / 平衡 / 扩展` 三档策略预设

但当前默认首屏仍然从 `核心层` 开始，因此用户初次打开面板时，通常仍主要看到：

- 反链块本身
- 文档级基本定位信息

这会导致一种很自然的感受：能力变强了，但第一眼不一定明显。

## 二、当前版本里，相邻节点如何看到

### 1. 当前规则

相邻节点已经纳入上下文体系，但它不属于默认首屏层级。

- 前相邻块：属于 `nearby`
- 后相邻块：属于 `nearby`
- 默认首屏：`core`

因此，若用户只是正常打开反链面板而不继续操作，通常不会直接看到相邻节点。

### 2. 当前正确操作方式

在当前版本中，查看相邻节点的操作方式如下：

1. 打开反链面板
2. 找到目标文档行
3. 单击文档标题文字区域
4. 文档上下文层级会按如下顺序递进：

   - `核心`
   - `近邻`
   - `扩展`
   - `全文`

第一次从 `核心` 切到 `近邻` 时，才会开始看到：

- 前相邻块
- 后相邻块
- 父级
- 部分子级

### 3. 注意区分的几个操作

当前交互里，几个动作是分开的：

- 左侧折叠箭头：折叠/展开文档行
- 单击文档标题：逐级展开上下文
- `Ctrl + 单击` 文档标题：直接打开对应反链块
- 左右箭头：切换同一文档内的不同反链

如果用户把这些动作混为一谈，就很容易误判为“逐级展示没有生效”。

## 三、当前版本的建议检查清单

若要验证“相邻节点是否已经生效”，建议按下面的顺序检查。

### 检查前提

优先选这种反链进行验证：

- 反链块位于普通段落中，而不是标题占位
- 反链块前后确实存在同级块
- 该文档内最好只有 1 条或少量反链，避免切错目标

### 检查步骤

1. 打开反链面板，先确认当前看到的是目标文档和目标反链
2. 观察首屏，只看到反链块本身是正常现象，因为默认还在 `核心层`
3. 单击文档标题文字，而不是左侧折叠按钮
4. 确认当前上下文已进入 `近邻层`
5. 检查是否出现前后相邻块、父级或子级内容
6. 若仍未出现，再继续单击标题进入 `扩展层`
7. 若还不确定，再切到 `全文`

### 若仍看不到相邻节点，优先排查

- 该反链本身没有前后同级块
- 当前显示的是同文档里的另一条反链
- 相邻块被预算裁剪
- 当前场景只有父级/子级可采集，没有有效相邻块
- 用户点的是折叠箭头或用了 `Ctrl + 单击`，而不是普通单击标题

## 四、逐级展示的当前交互问题

从用户体验看，当前逐级展示的主要问题不是“能力不存在”，而是“入口不显眼、结果不够可解释”。

### 1. 可发现性不足

当前逐级展开的主要入口是“单击文档标题”。  
这个动作本身并不直观，界面也没有明确提示“这里可以继续展开上下文”。

因此用户常见的真实体验是：

- 看到了折叠按钮，但不知道标题还能做另一种展开
- 以为标题只是普通跳转入口
- 以为面板里没有相邻节点能力

### 2. 缺少当前层级反馈

用户完成点击后，如果界面没有明确告诉他“现在已经从核心切到近邻”，用户只能靠肉眼猜。

这会带来两个问题：

- 不确定刚才点击是否生效
- 不知道下一次点击还会发生什么

### 3. 缺少“新增了哪些内容”的解释

即使切到了 `近邻`，用户也不一定能快速识别：

- 新出现的是前相邻块
- 新出现的是父级
- 新出现的是标题子级

如果上下文文本本身结构复杂，用户还是会觉得“只是多了一些内容”。

### 4. 折叠与展开的概念容易混淆

当前界面里同时存在：

- 文档折叠/展开
- 上下文逐级展开

这两个动作在模型上是不同的，但在界面上缺少清晰分离，容易让用户把它们理解成同一种“展开”。

## 五、交互优化建议

以下建议按优先级排序，优先解决“用户根本不知道怎么用”的问题，再解决“用了但不容易理解”的问题。

### P0：增加显式的上下文层级入口

最优先建议是在文档标题区域增加一个明确的“上下文层级按钮”或状态按钮。

建议表现形式：

- 按钮文案：`核心` / `近邻` / `扩展` / `全文`
- 或图标 + 文案：`上下文: 核心`
- 单击该按钮后逐级推进到下一层

这样做的价值：

- 用户能立刻知道“这里可以切换上下文层级”
- 不再依赖隐藏在标题点击里的手势
- 可把“折叠文档”和“展开上下文”明确拆开

### P0：显示当前层级状态

无论是否增加独立按钮，都建议始终在文档行显示当前层级状态。

建议示例：

- `当前：核心`
- `当前：近邻`
- `当前：扩展`
- `当前：全文`

这样用户至少能知道：

- 现在看到的是哪一层
- 再点一次会往哪一层前进

### P1：为标题点击增加明确提示

若仍保留“单击标题逐级展开”这一路径，建议至少增加显式提示。

建议方式：

- hover tooltip：`单击逐级展开上下文，Ctrl+单击打开块`
- 移动端长按或首次点击提示同类说明

这项改动成本低，但能明显降低误解。

### P1：切换后提示“新增展示内容”

建议在切到新层级后，短暂显示一条轻量说明，告诉用户这次多展示了什么。

示例：

- `已显示：前相邻块、后相邻块、父级`
- `已显示：标题子级与列表子级`
- `已进入全文模式`

这能把“内容变多”转化为“能力被理解”。

### P1：对预算裁剪增加提示

当 `nearby` 或 `extended` 已有更多内容，但因预算裁剪未全部显示时，应给出明确反馈。

建议提示：

- `部分上下文已裁剪，继续展开查看更多`
- `当前为精简预览，切换到扩展或全文查看更多`

否则用户会把“被裁剪”误解成“没有更多上下文”。

### P2：首次使用提供一次性引导

对首次使用反链面板或首次进入升级后版本的用户，可增加一次性提示：

- `单击标题可逐级查看近邻、扩展与全文上下文`

这不是长期依赖的主交互，但很适合作为升级期过渡。

### P2：命中标签与层级状态协同展示

当前已经有命中来源标签与命中摘要。后续可把它们与层级状态联动。

例如：

- `近邻 · 前相邻块`
- `近邻 · 父级`
- `扩展 · 标题子级`

这样用户既知道“命中了哪里”，也知道“这属于哪一层上下文”。

## 六、推荐的优化落地顺序

若后续要真正改交互，建议顺序如下：

1. 先补“当前层级可见”与“显式入口”
2. 再补“切换后的解释提示”
3. 再补“预算裁剪提示”
4. 最后考虑首次引导和更细的视觉优化

理由很简单：

- 没有显式入口，用户不会用
- 没有层级反馈，用户不理解
- 没有裁剪提示，用户会误判能力边界

## 七、结论

当前版本里，相邻节点能力已经接入，但它默认位于 `近邻层`，所以首屏通常不会直接出现。  
如果用户不知道“单击标题可逐级展开上下文”，就会自然产生“优化后看起来没变化”的感受。

因此，后续最值得优先优化的，不是继续增加更多上下文来源，而是把“逐级展示”从一个隐式能力，升级成一个：

- 可发现
- 可理解
- 可预期
- 可解释

的显式交互。
